01. Executive summary

What Pustakam is

Pustakam is an AI-powered book generation product designed to turn a topic, learning goal, or subject prompt into a structured, chapter-based learning book. It does not stop at draft generation. It also supports roadmap planning, module-by-module writing, reading mode, editing, analytics, study materials, backup, and export.

Input

A user describes what they want to learn or explain.

Process

The app plans the structure, writes chapters, and assembles the final book.

Output

The user receives a readable, editable, exportable learning asset.

Plain-language positioning

The best way to describe Pustakam is: "a personalized AI learning-book workflow." It behaves more like a compact publishing and study tool than a one-box chatbot.

What makes it stand out

  • Creates structure before long-form writing begins.
  • Shows visible progress during chapter generation.
  • Lets users pause, resume, retry, or recover work.
  • Provides a reader experience after creation, not just raw output.

Who it helps

  • Learners who want a tailored study book.
  • Teachers or trainers who need custom material quickly.
  • Creators building guides, manuals, or educational content.
  • Teams producing onboarding or knowledge documents.

Pustakam can be understood through three connected layers: composition, generation, and consumption.

03. End-to-end user journey

From first visit to finished book

The real user journey is straightforward, but the product hides a lot of complexity behind that simplicity.

1

Landing and discovery

The visitor enters through a branded landing page that explains the product, shows how it works, and offers sample outputs before asking for commitment.

2

Topic entry

The user types a topic or learning goal into the main composer. This can be short or detailed.

3

Prompt enhancement

The app can refine the topic using AI into a stronger brief with a clearer title, audience definition, and suggested learning structure.

4

Roadmap creation

The system generates a chapter roadmap before writing any long-form content. This acts as the structure for the rest of the project.

5

Module generation

The app writes modules one at a time while showing progress. The user can pause, resume, retry, or switch provider paths if necessary.

6

Assembly and post-generation use

After all modules are completed, the app assembles the final book and makes it available in reading mode, analytics view, and export formats.

04. Detailed walkthrough by screen

Major app areas

Landing page

Acts as the product front door. It includes positioning, a process view, sample books, and access points for account actions or getting started directly.

Home composer

Functions like a clean brief-building surface. The user enters the topic, then expands options if they want more control.

Advanced options

Exposes target audience, complexity, context, generation mode, and language without overloading the default experience.

Bookshelf

Stores generated projects as a searchable library, turning the app into a continuing workspace rather than a one-time interaction.

Project detail view

Shows title, goal, status, module count, word count, read-time estimate, and the full learning roadmap for the selected book.

Reading and export

Once complete, the book can be read, edited, analyzed, bookmarked, downloaded as Markdown, or exported as a professional PDF.

Important design principle

The product keeps the first step simple, then reveals control gradually. This is one of the strongest parts of the user experience.

05. Product systems and reliability

How the workflow stays stable

Pustakam is built so that the user does not lose all progress if generation becomes slow or fails halfway.

System behaviorWhat it means in practice
Checkpoint savingCompleted modules are saved during generation, so the user can continue later without starting from zero.
Pause and resumeThe user can stop a long run and return to it later, which matters for large books or interrupted sessions.
Retry logicWhen the app runs into rate limits or unstable responses, it can attempt recovery instead of simply failing silently.
Provider switchingIf one provider path is not working well, the app can offer an alternate route for the user to continue.
Local-first storageBooks, settings, and reading progress remain in local browser storage, so the app retains memory between sessions.
Cloud-backed accessWhen account features are enabled, authenticated requests can still route through the proper backend path without removing local utility.

What happens during writing

  • The roadmap defines the module sequence.
  • Each module is generated one by one.
  • Progress is shown inside the project view.
  • Completed modules contribute to the final word count and later assembly.

What happens after writing

  • The book is assembled into a single final output.
  • The app adds introduction, table of contents, summary, and glossary.
  • The user can read, edit, analyze, and export the result.
06. Reading, analytics, and export

Why the product feels complete after generation

The strongest part of Pustakam is that the experience continues after the book is written.

Reading mode

  • Supports light, sepia, and dark reading themes.
  • Allows font and size adjustments for readability.
  • Supports bookmarks and return-to-position behavior.
  • Includes an editing mode for refining the final text.

Analytics and support materials

  • Shows total word count and estimated reading time.
  • Provides topic extraction and difficulty classification.
  • Generates study-oriented downloads such as a summary and progress tracker.

Export capability

The final book can be downloaded as Markdown or exported as a structured PDF. The PDF path is more than basic print output. It is designed to produce a cleaner, more formal document with layout treatment and a disclaimer page.

Backup and restore

The settings area includes export and import for full-library backup, which is valuable for users who want portability or protection against browser resets.

07. Settings, account, and support surfaces

The surrounding product layers that make the app usable day to day

Beyond generation itself, Pustakam includes a number of supporting surfaces that make it feel like a real product environment instead of a single workflow screen.

Settings modal

The settings area covers theme preference, default generation mode, default language, backup actions, restore flow, and full local data reset.

Account flow

When authentication is enabled, users can sign in, create an account, and enter profile details such as role and learning interest during signup.

Subscription contact path

The app includes a lightweight subscription-contact route that moves the user toward WhatsApp or email rather than forcing a complicated purchase flow inside the interface.

Backup and data control

  • Export the local library and settings as a JSON archive.
  • Restore a prior library or merge imported books into the existing set.
  • Purge all stored data when a user wants a clean reset.

PWA and install behavior

  • Supports install prompts for a more app-like experience.
  • Tracks dismissals so the install prompt is not shown too aggressively.
  • Supports offline reading once content is already available locally.

Built-in help and trust surfaces

The product includes dedicated pages and modals for About, Terms, Privacy, Compliance, Disclaimer, API documentation, usage guide, and blog content. This matters because it gives the app clearer boundaries, better product education, and stronger trust signals for new users.

09. Intern briefing

What a new intern should understand quickly

If this document is being shared with interns, the most important thing is that they understand how to explain the product clearly and consistently.

What interns should remember

  • Pustakam is not just an AI writer. It is a structured learning-book workflow.
  • The strongest product story is the full journey: prompt, roadmap, chapters, reading, analytics, export.
  • The app is useful because it saves time while still giving the user control after generation.
  • The experience is strongest when shown through one complete example rather than abstract explanation.

What interns should notice in the app

  • The landing page positions the product before the user enters the workflow.
  • The composer keeps the first step simple and reveals options only when needed.
  • The product visibly manages long generation runs instead of hiding them.
  • The app has useful post-generation surfaces, which many AI tools do not.

Suggested 60-second product explanation for interns

"Pustakam helps someone turn a topic into a full learning book. The app first creates a roadmap, then writes the chapters, then packages everything into something the user can actually read, edit, analyze, and export. It is especially useful for people who want structured learning content fast, without building the whole thing manually."

Good first demo topic

Choose a topic that is practical, clear, and easy to judge, such as personal finance, startup basics, or beginner web development.

Good demo path

Show prompt entry, roadmap generation, chapter progress, then jump to a completed book to show reading mode and export capability.

Good closing line

End the walkthrough by showing that the result is not only generated, but also editable, readable, and shareable.

10. Messaging guardrails

What to emphasize, and what not to over-promise

This section is especially useful for interns or anyone speaking about the product publicly.

Do sayDo not say
"It creates structured learning books quickly.""It guarantees perfect accuracy on every topic."
"It helps users go from topic to roadmap to final export.""It replaces expert review in high-stakes fields."
"It is strong for study material, explainers, and custom guides.""It can be trusted blindly without checking the output."
"It includes editing, analytics, backup, and PDF export.""It is only useful because of the AI model."

Best practice for interns

Lead with workflow value, not hype. The strongest story is not "our AI is powerful." The strongest story is "the app gives people a structured, usable learning asset much faster than doing it manually."

11. FAQ

Short answers for quick reviews

Is it only a generator?

No. It includes planning, generation management, reading, editing, analytics, backup, and export.

Can users recover failed runs?

Yes. The app supports checkpoints, pause and resume behavior, retry logic, and alternate provider selection.

Can the output be edited?

Yes. Final book content can be opened in an editing mode and saved back into the project.

Does it work only online?

Generation depends on online access, but the app is designed with PWA behavior and offline reading utility after content exists locally.

What makes it different from a chatbot?

It keeps project structure, writes chapter-by-chapter, stores progress, and packages the result into a more usable document experience.

Is it suited for sharing?

Yes. The guide, export, and PDF behavior make it suitable for presenting output outside the app.
